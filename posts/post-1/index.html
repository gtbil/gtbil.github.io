Filter and Sort Your Graph Before plotting.

Posted on Sep 30, 2025

Once you have a graph from minigraph-cactus, the default odgi images can be misleading.

Make sure you sort your graph before plotting :-)

# extract only the relevant part of your graph from
# the `chrom-subproblems` directory
CHROM="chr1"

# convert to gfa format
vg convert -t 16 -W -f chrom-subproblems/${CHROM}/${CHROM}.vg > ${CHROM}.gfa

# then to odgi
odgi build -g ${CHROM}.gfa -o ${CHROM}.og -t 16

# extract the path names and prefixes for squishing together contigs from
# same assembly
odgi paths -i ${CHROM}.og -L -t 8 \
  | cut -f 1,2 -d '#' \
  | sort \
  | uniq > prefixes.tsv

# sort the odgi file
odgi sort --threads 16 --progress \
  -i ${CHROM}.og  \
  --path-sgd \
  --optimize \
  -o ${CHROM}.sort.og
  
# make the less confusing plot
odgi viz --threads 16\
  -i ${CHROM}.sort.og \
  -M ../prefixes.tsv \
  --color-by-mean-inversion-rate \
  -o ${CHROM}.sort.png
